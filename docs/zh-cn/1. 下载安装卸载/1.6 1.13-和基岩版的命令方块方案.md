# 1.6 1.13-和基岩版的命令方块方案

* 本内容仅提供方法 和TPA数据包文件本身事实上无关

## 双人方案

双人方案适用于双人联机的情况下, 直接使用`/trigger tpa`传送到对方

在开始搞命令方块之前 先新建以下记分项 （输入每行指令之前要加斜杠"/"）
```mcfunction
scoreboard objective add tpa trigger
```

设置以下命令方块: 
```mcfunction
tp @p[name="<玩家1>",scores={tpa=1..}] <玩家2> 
tp @p[name="<玩家2>",scores={tpa=1..}] <玩家1> 
```
第一行写在一个 循环 不受制约 保持开启 的命令方块中
第二行写在一个 连锁 条件制约 保持开启 的命令方块中
两个命令方块的朝向需要保持一致且后者需要在前者方块材质侧面三角形的指向方向
摆放如图所示
![命令方块示意](docs/assets/1.6_command_blocks.png)
其中的尖括号"<>"包括的部分包括尖括号本身需要用真实的玩家名替代

## 多人方案

多人方案适用于多人方案的情况下, 直接使用`/trigger tpa set <ID>`传送到对应ID的玩家

在开始搞命令方块之前 先新建以下记分项 （输入每行指令之前要加斜杠"/"）
```mcfunction
scoreboard objective add tpa trigger
scoreboard objective add id dummy
```

设置以下命令方块: 
```mcfunction
tp @p[name="<玩家1>",scores={tpa=1..}] <玩家2> 
tp @p[name="<玩家2>",scores={tpa=1..}] <玩家1> 
```
第一行写在一个 循环 不受制约 保持开启 的命令方块中
第二行写在一个 连锁 条件制约 保持开启 的命令方块中
两个命令方块的朝向需要保持一致且后者需要在前者方块材质侧面三角形的指向方向

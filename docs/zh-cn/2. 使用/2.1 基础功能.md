# 2.1 基础功能

使用 `/trigger tpa` 或者 在进入游戏后点击左下角的`[点我打开TPA菜单]` 来打开TPA菜单

通过/trigger tpa这一指令来打开TPA菜单 无论什么情况 一定要记住这条唯一需要记住的指令

当一个请求被发送 请求者可以选择在对方接受前取消 未开启自接受的被请求者可以在请求超时前选择请求或者拒绝

请求的接受可以选择点击聊天栏中的接受 或者手动输入/trigger tpaccept来接受请求

当请求超时 请求者的取消和被请求者的接受/拒绝均会无效并显示

超时时间则可以在设置界面中进行调整 单位为游戏刻 默认为6000 即流畅情况下的30分钟 可以通过设置0 和-1 分别达到禁止传送和永不超时的效果

需要注意的一点是请求是和请求玩家绑定的 换句话说 每个玩家只允许同时持有一个请求（无论是传送请求还是传送此处请求）当另一个请求被发送前一个请求将被取消

轻量版中不存在传送请求 接受 取消 超时等概念 在轻量版中玩家在请求传送后便被立即传送到被请求玩家处

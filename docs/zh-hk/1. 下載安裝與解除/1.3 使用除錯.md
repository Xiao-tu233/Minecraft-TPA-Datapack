# 1.3 使用及除錯

本節會引導你逐步排查問題，以確認 TPA Datapack 已正確安裝並完成初始化。

---

## 一、確認 Datapack 是否已正確載入

請先喺遊戲內執行 `/datapack list`，查看列表中係咪存在 **TPA_Datapack**。

### 如果冇顯示 TPA_Datapack

通常代表 Datapack 未有正確安裝，請檢查以下內容：

* 所下載嘅 Datapack 支援嘅 **遊戲版本範圍** 係咪包含目前伺服器或存檔所使用嘅遊戲版本
  （注意：伺服器版本未必同客戶端版本相同）
* Datapack 係咪已放入**正確嘅存檔資料夾**

  * 如果伺服器安裝咗 Mod，可以由 Mod 資料夾返回上一層，通常就係伺服器根目錄
  * 部分伺服器可能修改咗預設存檔名稱，可以透過檢查
    `server.properties` 入面嘅 `level-name` 設定，確認實際使用嘅存檔資料夾名稱
  * 對於單人世界，建議先儲存並離開世界，再喺單人遊戲畫面選擇最近儲存嘅世界，
    撳下方嘅 `編輯…… -> 打開世界存檔文件夾` 按鈕即可定位

---

## 二、確認核心函數是否正常運作

如果 **TPA_Datapack** 已存在，請繼續執行 `/trigger tpa`，以檢查核心函數係咪正常運作。

### 顯示「未知嘅記分項目 'tpa'」

代表 **load 函數未有正常執行**。

處理方法如下：

* 喺遊戲內執行一次 `/reload`
* 或直接執行 `/function tpa:load`

通常喺伺服器重新啟動或 Datapack 重新載入後，都需要確保 load 函數至少成功執行一次。

---

### 顯示「你暫時無法觸發此記分項目」

通常代表 **tick 函數未有正常運作**，或者目前 tick 被跳過。

請依照以下順序排查：

#### 檢查執行頻率設定

* 執行 `/function tpa:options`
* 檢查 Datapack 嘅執行頻率係咪被設定得過低
* 建議恢復為預設值 **20**

#### 完整版特有情況（1.15 – 1.20.1）

喺部分版本嘅完整版中，如果某個傳送流程異常地持續執行，亦有可能導致 tick 無法正常結束。

呢個情況下可以考慮：

* 手動將傳送執行者傳送到**預期終點附近**
  ，令終點錨點可以被正確偵測，從而結束傳送狀態

可以透過以下方式確認相關資訊：

* 使用 `/data get storage tpa:tpa temp.teleport` 查看預期終點座標

* 正在執行傳送嘅玩家會帶有 `tpa.teleport` 標籤

* 範例（以 1.20.1 為例）

假設透過 data get 指令取得以下資訊：

```mcfunction
{ x: 1024.5d, y: 64.0d, z: 13.0d, dim: "minecraft:overworld", ... }
```

則可以執行：

```mcfunction
execute in minecraft:overworld run tp @p[tag=tpa.teleport] 1024 64 13
```

---

#### tick 標籤函數完全冇執行嘅情況

如果確認以上情況都唔適用，而 tick 相關標籤函數仍然未有正常運作：

* 請使用擁有 **OP 權限** 嘅玩家客戶端執行
  `/function tpa:tick_not_working`，或者撳設定選單中嘅 `[無法運行]` 按鈕
* 並根據函數提供嘅提示進行處理

---

由於實際環境同情況可能比較複雜，

如果問題仍然未能解決，歡迎加入交流群進一步查詢。

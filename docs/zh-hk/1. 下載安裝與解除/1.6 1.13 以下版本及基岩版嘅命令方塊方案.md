# 1.6 1.13 以下版本及基岩版嘅命令方塊方案

* 本節內容僅提供實現思路及方法參考，與 TPA Datapack 檔案本身並無直接關聯
* 對於 1.9–1.12.2，`/tag` 指令可能需要改用 `/scoreboard players tag` 替代

---

## 雙人方案

### 最終效果

雙人方案適用於只有兩名玩家連線嘅情況。
玩家可以直接使用 `/trigger tpa` 傳送到對方所在位置。

### 手動輸入部分

喺放置命令方塊之前，請先建立以下記分項目。
（注意：輸入每條指令前都需要加上斜線「/」）

```
scoreboard objectives add tpa trigger
```

### 命令方塊部分

請依照以下順序設定命令方塊：

```
scoreboard players enable @a tpa
tp @p[name="<玩家1>",scores={tpa=1..}] <玩家2>
tp @p[name="<玩家2>",scores={tpa=1..}] <玩家1>
scoreboard players set @a tpa 0
```

* 第一行放入 **循環／無條件／保持開啟** 嘅命令方塊
* 第二至第四行分別放入 **三個連鎖／無條件／保持開啟** 嘅命令方塊

其中尖括號「<>」及其內容需要整體替換成實際玩家名稱。

命令方塊放置示意如下：
![命令方塊示意](/docs/assets/img/1.6_command_blocks.png)

## 多人方案

### 最終效果

多人方案適用於多人連線環境。
每位玩家進入世界後都會由系統分配一個唯一 ID。

如果需要查看目前玩家 ID，可以執行：

* `/scoreboard objectives setdisplay sidebar id`
* 查看完成後使用 `/scoreboard objectives setdisplay sidebar` 關閉顯示

玩家可以透過 `/trigger tpa set <ID>` 傳送到對應 ID 嘅玩家。

### 手動輸入部分

喺放置命令方塊之前，請依次輸入以下指令：

```
scoreboard objectives add tpa trigger
scoreboard objectives add id dummy
scoreboard players set #global_current id 0
```

---

### 命令方塊部分

#### 1. 分配 ID

請按照下圖所示方式放置命令方塊（建議確保圖片方向為「上北下南」）：
![命令方塊示意](/docs/assets/img//1.6_command_blocks_.png)

**命令方塊 A**：循環／無條件／保持開啟

```
execute as @a unless score @s id = @s id run scoreboard players set @s id 0
```

**命令方塊 B**：循環／無條件／保持開啟

```
execute as @p[scores={id=0}] run scoreboard players add #global_current id 1
```

**命令方塊 C、D**：緊接放置，連鎖／有條件／保持開啟

```
tellraw @a ["[TPA] ", {"selector": "@p[scores={id=0}]"}, " 獲分配咗 ID：", {"score": {"name": "#global_current", "objective": "id"}}]
scoreboard players operation @p[scores={id=0}] id = #global_current id
```

---

#### 2. 處理傳送

傳送邏輯提供兩種方案：

* **自閉鏈方案**：
  實現較複雜，但理論上可以支援無限玩家

* **枚舉方案**：
  實現較簡單，但只適用於玩家數量有限嘅情況

---

##### 自閉鏈方案

⚠ 本方案對命令方塊嘅空間配置及朝向要求非常高。
錯誤放置或修改 **有可能導致存檔損壞甚至崩潰**，請務必嚴格按照步驟操作。

最終命令方塊布局如下（圖中混凝土僅作尺寸參考）：
![命令方塊示意](/docs/assets/img//1.6_command_blocks___.png)

喺命令方塊 A 南側放置、朝向西嘅 **命令方塊 E**：
循環／無條件／**紅石控制**

```
scoreboard players enable @a tpa
```

之後沿命令方塊 E 向西放置 **F、G、H、I、J、K、L、M₀、N₀**
全部設為：連鎖／無條件／保持開啟

```
tp @p[tag=reqer] @p[tag=target]
scoreboard players set @a[tag=reqer] tpa 0
tag @a remove reqer
tag @a remove target
tag @a remove notmatch
scoreboard players set #found tpa 0
tag @p[scores={tpa=1..}] add reqer
tag @p[tag=!notmatch] add target
execute if score @p[tag=target] id = @p[tag=reqer] tpa run scoreboard players set #found tpa 1
```

繼續喺 N₀ 西側放置、朝向南嘅 **命令方塊 O₀**：
連鎖／無條件／保持開啟

```
execute if score #found tpa matches 0 run tag @p[tag=target] add notmatch
```

之後喺 O₀ 南側放置、朝向東嘅 **命令方塊 P₀**：
連鎖／無條件／保持開啟

```
execute if score #found tpa matches 0 run tag @a remove target
```

繼續喺 P₀ 東側放置 **命令方塊 Q₀**（毋須填寫指令），
再喺 Q₀ 東側放置朝向北嘅 **命令方塊 R₀**：

```
execute if score #found tpa matches 0 if entity @p[tag=!notmatch] run clone ~-2 ~ ~-3 ~ ~ ~-2 ~-2 ~ ~-1
```

最後，玩家站喺 **O₀（第一個轉角位、西北角嘅命令方塊）正中央**，
喺聊天欄輸入：

```
clone ~ ~-1 ~ ~2 ~-1 ~1 ~ ~-1 ~-2
```

完成後，身後會生成 **M₁、N₁、O₁、P₁、Q₁、R₁** 六個命令方塊。

##### 枚舉方案

```
scoreboard players enable @a tpa
tp @a[scores={tpa=1}] @p[scores={id=1}]
tp @a[scores={tpa=2}] @p[scores={id=2}]
tp @a[scores={tpa=3}] @p[scores={id=3}]
...
scoreboard players set @a tpa 0
```

枚舉方案嘅邏輯非常直觀，只需要按照上述模式繼續補充即可。

* 如果命令數量過多，可以由尾部開始適當裁減
* 如果需要支援更多玩家，可以繼續向下擴展
* 當玩家數量較多時，仍然建議優先使用自閉鏈方案（雖然配置較複雜）

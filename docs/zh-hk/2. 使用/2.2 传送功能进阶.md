# 2.2 傳送功能進階

## 自動接受

喺完整版中，玩家可以喺 TPA 選單入面，或者使用指令 `/trigger tpaccept set 2` 開啟自動接受功能。

開啟後，同樣可以透過選單，或者使用指令 `/trigger tpaccept set 3` 將其關閉。

開啟自動接受後，系統會自動接受任何玩家向自己發送嘅傳送請求，但「傳送到此處」請求仍然需要手動處理。

## 傳送到此處

傳送到此處（tpahere）係一種傳送方式，由請求者向目標玩家發送請求，令對方傳送到請求者目前所在位置。

要發送傳送到此處請求，需要開啟「傳送到此處」選單。佢同一般傳送選單睇落好相似，但上方嘅文字提示會有所不同。

喺傳送到此處選單中，同普通傳送一樣，直接撳玩家名稱即可發送傳送請求。

傳送到此處嘅傳送目標由記分項目 `tpa.tpahere` 控制。

| 版本     | 內容                                                                                                |
| :----- | :------------------------------------------------------------------------------------------------ |
| 1.7 以前 | 加入自動接受功能                                                                                          |
| 1.9    | 加入傳送到此處請求                                                                                         |
| 2.0    | 自動接受改由記分項目 `tpa.tpaccept_toggle`（`tpa.tpaccept_tgl`）控制                                            |
| 2.0.6  | 傳送到此處請求嘅目標而家唔再儲存於記分項目 `tpa.tp_here`，而係改為儲存於 `tpa.tp_to`；原本嘅記分項目則改為作為布林值，用嚟儲存目前發送嘅請求係傳送請求定係傳送到此處請求 |
